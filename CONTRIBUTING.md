# Contributing to Legal Case Researcher

Thank you for your interest in contributing to **Legal Case Researcher**! This project is part of the [90 Local LLM Projects](https://github.com/kennedyraju55/90-local-llm-projects) collection.

## Getting Started

1. **Fork** the repository
2. **Clone** your fork locally
3. **Create a branch** for your feature or fix

```bash
git checkout -b feature/my-improvement
```

4. **Install dependencies**

```bash
pip install -r requirements.txt
```

5. **Run tests** to make sure everything works

```bash
python -m pytest tests/ -v
```

## Development Guidelines

- Follow PEP 8 style guidelines
- Add type hints to all function signatures
- Write docstrings for all public functions
- Add tests for new features
- Keep the privacy-first architecture — no external API calls

## Pull Requests

- Describe your changes clearly
- Reference any related issues
- Ensure all tests pass
- Keep commits focused and well-described

## Code of Conduct

Be respectful, inclusive, and constructive. We're all here to build great tools.

## Questions?

Open an issue or start a discussion in the repository.
